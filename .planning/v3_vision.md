# v3.0 Vision: FTT-Style Coaching Diagnostic System

## 核心转变
从"AI 出分析报告"变成"教练看视频后的诊断指导"

## FTT 的风格是什么
FTT 的 Johnny 看学员视频时：
1. 不是列 10 个问题，而是找到 THE ONE THING
2. 解释为什么这一个问题导致了你看到的所有表面现象
3. 用生物力学原理解释 WHY（不是现象描述）
4. 给一个具体的 drill，带感觉提示
5. 语气专业但温和，像在面对面教学

## 两条知识路径交叉
1. **教练视角（知识库）**：732 概念 → 诊断链 → "你的问题是 X，因为 Y 肌群没参与"
2. **学员视角（learning.md）**：22 次训练 → 突破/瓶颈 → "你 4/1 解决了动力链串通，但今天视频显示连接又断了"

两者交叉 = 个性化的、有历史感的教练诊断

## 系统需要做什么
1. **深挖 learning.md**：提取问题演化链（哪些问题反复出现？哪些已解决？当前卡点在哪？）
2. **VLM prompt 重构**：从"列问题"格式变成"找一个关键点，深入解释"格式
3. **报告重构**：从条目式变成叙事式，像教练在说话
4. **知识图谱利用**：VLM 看到症状 → 图谱追溯根因 → 结合用户历史 → 给出个性化建议
5. **验证**：用现有视频重新分析，确保输出质量达到 FTT 水平
