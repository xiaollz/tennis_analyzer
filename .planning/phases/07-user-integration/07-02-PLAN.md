Create `knowledge/output/training_plan.py`:

1. **Models**:
   - `DrillRecommendation(BaseModel)`: drill_id (str), drill_name (str), drill_name_zh (str), addresses (list[str] — concept IDs), priority (int), rationale (str)
   - `TrainingPlan(BaseModel)`: generated_date (str), focus_areas (list[str] — concept names), drills (list[DrillRecommendation]), summary (str), summary_zh (str)
   - Method: `to_markdown() -> str` — renders as readable Markdown with Chinese headers

2. **generate_training_plan(profile: UserProfile, graph: KnowledgeGraph, chains: list[DiagnosticChain]) -> TrainingPlan**:
   - Get active_issues from profile (struggling + regressed)
   - For each issue concept_id, find drills via:
     a. Direct `drills_for` edges in graph pointing to that concept
     b. Walk causal chain from concept to find ancestor concepts that have `drills_for` edges
     c. Check diagnostic chains — if issue matches a chain's root_cause, use the chain's drills list
   - Score each drill: +2 per struggling concept addressed, +3 per regressed concept (regression = higher priority)
   - Sort by score descending
   - Build rationale string per drill: "Addresses: {concept_names}. {chain_context if applicable}"
   - If no active issues, return plan with summary "All tracked concepts are mastered or improving. Continue current training."
