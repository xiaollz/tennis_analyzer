Create `knowledge/user_profile.py` with:

1. **Pydantic models** (follow existing schemas.py pattern):
   - `ConceptStatus(str, Enum)`: struggling, improving, mastered, regressed
   - `ConceptLink(BaseModel)`: concept_id (str), status (ConceptStatus), first_seen (str, date), last_seen (str, date), cues (list[str])
   - `SessionEntry(BaseModel)`: date (str), summary (str), concept_links (list[ConceptLink]), breakthroughs (list[str])
   - `UserProfile(BaseModel)`: sessions (list[SessionEntry]), concept_map (dict[str, ConceptLink])
   - Methods on UserProfile: `get_status(concept_id)`, `active_issues()`, `recent_breakthroughs(n)`, `to_json(path)`, `from_json(path)`

2. **build_profile_from_learning(learning_path, registry_snapshot_path)** function:
   - Parse learning.md by `## YYYY-MM-DD` section headers
   - For each section, extract:
     - Bold terms (`**text**`) as candidate concept names
     - Terms from causal chain code blocks (` ``` ` sections with `→` arrows)
     - Cue strings from numbered lists or backtick-quoted text
   - Resolve each candidate term against registry using `ConceptRegistry.resolve(term, threshold=65)` — use lower threshold (65) because Chinese terms need more flexibility
   - Also try resolving the English equivalent if a known mapping exists (e.g., "scooping" for "捞球")
   - Derive status per concept across sessions:
     - First mention as problem = struggling
     - Later mention as "solved/mastered/breakthrough" = mastered
     - Mentioned as problem again after mastery = regressed
     - Positive trend without explicit mastery = improving
   - Status derivation heuristic: scan for positive keywords (突破, 成功, mastered, 解决) vs negative (问题, 仍然, 还是, still) near concept mentions

3. **Tests** in `tests/test_user_profile.py`:
   - Unit tests for models (construction, serialization)
   - Test parsing of a minimal learning.md snippet (3 sessions)
   - Test fuzzy resolve integration with a small mock registry
   - Test status progression logic
