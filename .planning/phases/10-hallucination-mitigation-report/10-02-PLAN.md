# Phase 10 Plan 2: Diagnostic Journey Report Section

## Objective
Add a "diagnostic journey" narrative section to the report generator that shows the iterative reasoning process (hypotheses tested, evidence found, eliminations) as a readable narrative, while preserving backward compatibility with the v1.0 root_cause_tree JSON format.

## Context
- @report/report_generator.py — _vlm_section, _format_root_cause_tree
- @knowledge/schemas.py — DiagnosticSession model
- @.planning/research/V2_RESEARCH.md — Section 8: Presenting Iterative Reasoning

## Tasks

### Task 1: Diagnostic Journey Narrative Generator (RG-01, RG-02)
type="auto"

**Action:**
1. Add `_format_diagnostic_journey()` static method to ReportGenerator:
   - Input: DiagnosticSession (or equivalent dict with rounds, hypotheses, observations)
   - Output: List[str] of Markdown lines
   - For each round: summarize what was observed and what changed
   - Use narrative style: "First I noticed..." / "This led me to suspect..." / "After checking X, I confirmed Y"
   - Chinese language output matching existing report style
2. Each round section includes:
   - Round purpose (scan / targeted observation / confirmation)
   - Key observations made (frame + description, not raw JSON)
   - Hypothesis changes (confirmed/eliminated with reason)
   - Transition to next round reasoning
3. Keep the narrative concise: max ~200 chars per round summary

**Done when:**
- `_format_diagnostic_journey()` produces readable Chinese narrative
- A 3-round session renders as 3 narrative paragraphs
- Unit test validates output structure and content

### Task 2: Integration into Report + Backward Compatibility (RG-03, RG-04)
type="auto"

**Action:**
1. Modify `_vlm_section()` to detect multi-round results:
   - Check for `diagnostic_session` key in vlm_result dict
   - If present: render diagnostic journey section BEFORE root_cause_tree
   - If absent: existing behavior unchanged (backward compat)
2. Add `include_journey` flag (default True) to control journey section visibility (RG-04)
3. Update `_build_final_result()` in MultiRoundAnalyzer to include `diagnostic_session` key with serialized session data for the report generator
4. Ensure root_cause_tree JSON schema is preserved in multi-round output (RG-03)
5. Add journey section header: "### 诊断推理过程" with collapsible hint

**Done when:**
- Multi-round vlm_result with diagnostic_session renders journey + root_cause_tree
- v1.0 vlm_result without diagnostic_session renders exactly as before
- include_journey=False suppresses journey section
- root_cause_tree JSON unchanged between v1.0 and v2.0
- Unit tests cover all 3 scenarios
