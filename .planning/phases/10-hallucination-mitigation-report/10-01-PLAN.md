# Phase 10 Plan 1: Hallucination Mitigation

## Objective
Add four hallucination mitigation mechanisms to the multi-round VLM diagnostic loop: observation anchoring validation, cross-round contradiction detection, quantitative YOLO cross-validation, and low-confidence re-observation triggers.

## Context
- @knowledge/schemas.py — Observation model (add anchoring validation)
- @evaluation/vlm_analyzer.py — MultiRoundAnalyzer (add hallucination checks)
- @.planning/research/V2_RESEARCH.md — Section 5: Handling VLM Hallucinations

## Tasks

### Task 1: Observation Anchoring Validation (HM-01)
type="auto"

**Action:**
1. Add `validate_anchoring()` method to MultiRoundAnalyzer that checks each Observation:
   - Must have a non-empty `frame` field matching pattern "图N" or "Frame N"
   - Must have `description` length >= 10 chars (not just "yes"/"no")
   - Flag unanchored observations by setting confidence to 0.0 and marking as re-observe candidates
2. Add `_reobserve_unanchored()` that builds a targeted re-prompt for unanchored observations
3. Call `validate_anchoring()` after parsing observations in `_execute_diagnostic_round()`
4. Add Observation model validator (Pydantic field_validator) for `frame` field — warn but don't reject empty frames

**Done when:**
- Observations without frame references get confidence set to 0.0
- A method exists to generate re-observation prompts for unanchored observations
- Unit tests pass for anchoring validation

### Task 2: Cross-Round Contradiction Detection (HM-02)
type="auto"

**Action:**
1. Add `_detect_contradictions()` method to MultiRoundAnalyzer:
   - Compare observations across rounds that reference the same frame + same visual feature
   - If judgment flipped (yes->no or no->yes) on same feature, flag as contradiction
   - Return list of contradiction pairs
2. Add `_resolve_contradictions()` that:
   - For each contradiction: build a highly specific re-observation prompt mentioning both prior observations
   - Set contradicted observations' confidence to 0.3 (uncertain)
   - If a 3rd observation still contradicts, fall back to quantitative data (Task 3)
3. Integrate into the round loop: after scoring, before convergence check

**Done when:**
- 3 synthetic contradiction scenarios tested and detected correctly
- Contradictions trigger re-observation in next round
- Unit tests cover: same-frame contradiction, different-frame no-trigger, triple-contradiction fallback

### Task 3: Quantitative Cross-Validation (HM-03)
type="auto"

**Action:**
1. Add `_cross_validate_with_kinematics()` method to MultiRoundAnalyzer:
   - Accept supplementary_metrics dict (elbow_angle, wrist_trajectory, shoulder_rotation)
   - For observations about elbow angle: compare VLM judgment vs actual measured angle
   - For observations about wrist: compare trajectory shape description vs measured wrist path
   - Override VLM observation if quantitative data clearly contradicts (>30 degree discrepancy for angles)
2. Store supplementary_metrics on the session so it's available across rounds
3. Add `supplementary_metrics` field to DiagnosticSession model
4. Call cross-validation after observation scoring in each round

**Done when:**
- VLM "elbow wide" overridden when YOLO measures <90 degrees
- VLM "wrist drops sharply" overridden when trajectory is smooth
- Override logged with reason in observation metadata
- Unit tests cover override and no-override scenarios

### Task 4: Low-Confidence Re-observation Triggers (HM-04) + Integration
type="auto"

**Action:**
1. Add `_collect_reobserve_candidates()` to MultiRoundAnalyzer:
   - Observations with confidence < 0.5 are candidates
   - Unanchored observations (from Task 1) are candidates
   - Contradicted observations (from Task 2) are candidates
2. Modify `_execute_diagnostic_round()` to include re-observation directives when candidates exist
3. Add `reobserve_candidates` field to DiagnosticSession to track across rounds
4. Ensure per-round prompts stay under 4K chars (CP-03): truncate re-observation list if needed

**Done when:**
- Low-confidence observations appear in next round's prompt as re-observation items
- Re-observation candidates tracked in session state
- Prompt size checked and truncated if over 4K for non-final rounds
- All 4 hallucination checks work together in integration test
