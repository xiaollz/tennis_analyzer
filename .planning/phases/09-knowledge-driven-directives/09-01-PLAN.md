# Phase 9 Plan 1: Observation Directive Templates + compile_observation_directive()

## Objective
Add Jinja2 templates and VLMPromptCompiler methods that convert active hypotheses + unchecked diagnostic steps into targeted per-round VLM observation prompts, plus a final confirmation template.

## Context
- @knowledge/output/vlm_prompt.py — VLMPromptCompiler (add compile_observation_directive, compile_confirmation_prompt)
- @knowledge/templates/vlm/ — existing templates (add observation_directive.j2, confirmation.j2)
- @knowledge/schemas.py — Hypothesis, DiagnosticChain, DiagnosticStep models
- @.planning/research/V2_RESEARCH.md — template design spec

## Tasks

### Task 1: Create observation_directive.j2 template
type="auto"

Create `knowledge/templates/vlm/observation_directive.j2` that renders:
- Current round number and hypothesis context (name_zh, confidence)
- Per-directive observation questions (frame, question_zh, vlm_features)
- Strict JSON response format (observations + hypothesis_updates)

Budget: < 4K chars rendered for typical 2-3 hypothesis scenario.

**Done when:** Template file exists and can be rendered by Jinja2 with sample data.

### Task 2: Create confirmation.j2 template
type="auto"

Create `knowledge/templates/vlm/confirmation.j2` for the final confirmation round:
- Full evidence summary (all observations grouped by hypothesis)
- Active/confirmed hypotheses with confidence
- Request for root_cause_tree JSON output (v1.0 compatible format)
- Causal chain summary

Budget: < 6K chars rendered.

**Done when:** Template file exists and can be rendered by Jinja2 with sample data.

### Task 3: Add compile_observation_directive() and compile_confirmation_prompt() to VLMPromptCompiler
type="auto"

Add two new methods to VLMPromptCompiler:

1. `compile_observation_directive(session, round_number)` — generates targeted observation prompt from:
   - Active hypotheses in session
   - Unchecked diagnostic steps (session.checked_steps vs chain.check_sequence)
   - Concept.vlm_features for relevant concepts
   - DiagnosticChain.vlm_frame for frame targeting
   - Maps DiagnosticStep.check into specific VLM questions (KD-04)

2. `compile_confirmation_prompt(session)` — generates final round prompt from:
   - All observations grouped by hypothesis
   - Confirmed + active hypotheses
   - Root cause tree generation instruction

Both methods enforce character budgets (4K for directive, 6K for confirmation).

**Done when:** Both methods exist, render valid prompts, and stay within budget.

### Task 4: Tests for templates and compiler methods
type="auto"

Add tests in `tests/test_vlm_prompt.py`:
- Test observation_directive.j2 renders with mock data, contains Chinese hypothesis context
- Test confirmation.j2 renders with mock data, contains evidence summary
- Test compile_observation_directive() generates < 4K prompt with hypothesis-specific questions
- Test compile_confirmation_prompt() generates < 6K prompt with evidence summary
- Test KD-04: directive maps DiagnosticStep.check + vlm_features + vlm_frame into VLM questions
- Test empty hypotheses edge case (no active hypotheses -> minimal prompt)

**Done when:** All tests pass with `pytest tests/test_vlm_prompt.py`.
