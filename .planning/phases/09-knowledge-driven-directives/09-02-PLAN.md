# Phase 9 Plan 2: Hypothesis Tracker + Cross-Causal Reasoning + MultiRoundAnalyzer Integration

## Objective
Implement hypothesis lifecycle management (created -> active -> confirmed/eliminated), confidence scoring from observations, cross-hypothesis causal reasoning via get_causal_chain(), and integrate everything into MultiRoundAnalyzer so each round uses knowledge-driven directives and progressive narrowing.

## Context
- @evaluation/vlm_analyzer.py — MultiRoundAnalyzer (enhance _execute_diagnostic_round, add cross-hypothesis reasoning)
- @knowledge/graph.py — KnowledgeGraph.get_causal_chain() (use for cross-hypothesis reasoning)
- @knowledge/output/vlm_prompt.py — compile_observation_directive() from Plan 1
- @knowledge/schemas.py — Hypothesis, Observation, DiagnosticSession

## Tasks

### Task 1: Implement HypothesisTracker class
type="auto"

Create hypothesis tracking logic (can be a class or methods on MultiRoundAnalyzer):

1. **Confidence scoring (HT-02):** Each observation adjusts hypothesis confidence:
   - Supporting observation (judgment=yes, high confidence): +0.15
   - Contradicting observation (judgment=no, high confidence): -0.20
   - Unclear observation: no change
   - Scale by observation confidence (multiply delta by obs.confidence)
   - Clamp to [0.0, 1.0]
   - Auto-eliminate when confidence drops below 0.15
   - Auto-confirm when confidence rises above 0.85

2. **Cross-hypothesis reasoning (HT-03):** After each round:
   - For each confirmed hypothesis, call get_causal_chain() to find downstream symptoms
   - If another active hypothesis's symptom_concept_id appears as downstream of a confirmed root cause, auto-eliminate it and mark as downstream symptom
   - Record the causal link in hypothesis update reason

3. **Progressive narrowing (HT-04):** Track that each round eliminates or confirms at least one hypothesis. Log warning if a round makes no progress.

**Done when:** Confidence scoring adjusts hypothesis confidence based on observations, cross-hypothesis reasoning auto-eliminates downstream symptoms, progressive narrowing is tracked.

### Task 2: Integrate into MultiRoundAnalyzer
type="auto"

Modify MultiRoundAnalyzer to use knowledge-driven directives:

1. Replace `_execute_diagnostic_round()` to use `compile_observation_directive()` instead of `compile_pass2_prompt()` for rounds 1..N-1
2. Use `compile_confirmation_prompt()` for the final round (when convergence detected or last round)
3. Apply confidence scoring to observations (from Task 1)
4. Run cross-hypothesis reasoning after each round's updates
5. Track checked_steps in session as directives are generated
6. Link observations to hypotheses (populate supporting_observations / contradicting_observations)
7. Pass the KnowledgeGraph reference to MultiRoundAnalyzer (needed for causal chain queries)

**Done when:** MultiRoundAnalyzer uses observation directives for intermediate rounds, confirmation template for final round, and applies hypothesis tracking logic.

### Task 3: Tests for hypothesis tracking and integration
type="auto"

Add/enhance tests in `tests/test_multi_round.py`:
- Test confidence increases on supporting observation
- Test confidence decreases on contradicting observation
- Test auto-eliminate when confidence < 0.15
- Test auto-confirm when confidence > 0.85
- Test cross-hypothesis reasoning: confirmed A causes B -> B auto-eliminated
- Test progressive narrowing: round makes progress
- Test full run() uses compile_observation_directive (not compile_pass2_prompt) for diagnostic rounds
- Test final round uses compile_confirmation_prompt

**Done when:** All tests pass with `pytest tests/test_multi_round.py`.
