    Create `knowledge/pipeline/handlers.py` with handler functions. Each handler reads a Markdown file and returns an ExtractionResult (concepts + edges).

    **Key principle:** Handlers do NOT create new concepts from thin air. They match text against the seeded registry (via registry.resolve()) and extract RELATIONSHIPS. New concepts are only created when clearly not matching anything in the registry. This prevents concept explosion.

    **Handler implementations:**

    1. `extract_synthesis(filepath, registry)` -- For files 12, 13, 15, 17. These are the most structured. Parse headers as concept names, resolve against registry, extract cross-references as edges. New concepts found here ARE added to registry (synthesis files establish canonical names).

    2. `extract_ftt_book(filepath, registry)` -- For files 01, 02. Parse section headers, look for concept definitions in prose. Match against registry. Extract causal relationships from arrow notation (A -> B -> C).

    3. `extract_ftt_blog(filepath, registry)` -- For files 04-08. Similar to ftt_book but with more structured analysis sections. Parse "key concepts", "key principles" sections.

    4. `extract_ftt_videos(filepath, registry)` -- For files 09_ftt_videos_*. Per-video sections with headers. Extract UNIQUE concepts only (per RESEARCH.md Pitfall 5). Use synthesis as primary, per-video only for NEW concepts.

    5. `extract_tpa_videos(filepath, registry)` -- For files 14_*, 16_*. Same approach as ftt_videos.

    6. `extract_ftt_specific(filepath, registry)` -- For files 18-23. Each is a deep dive on one concept. The concept itself should already be in registry; extract supporting details, aliases, edges.

    7. `extract_biomechanics(filepath, registry)` -- For files 24-28. **Critical:** Parse muscle tables using regex (look for | delimiters with muscle-related headers). Create concepts with `muscles_involved` populated. Extract FTT mapping cross-references ("> **FTT映射：**" pattern) as `supports` edges. File 28 has problem-solution mappings -> create `fixes` edges.

    8. `extract_generic(filepath, registry)` -- Fallback for file 03. Simple header + keyword extraction.

    **Edge extraction patterns (apply across all handlers):**
    - Arrow chains: `re.split(r'\s*(?:→|->|==>)\s*', line)` -> CAUSES edges
    - FTT mapping: `r'FTT映射.*?`(.+?)`'` -> SUPPORTS edges  
    - "Related concepts" / "See also" sections -> SUPPORTS edges
    - Problem-solution pairs -> DRILLS_FOR edges

    **Edge confidence by source type:**
    - Synthesis files: 0.9
    - Primary sources (book, blog): 0.8
    - Per-video analysis: 0.6
    - User journey: 0.7
