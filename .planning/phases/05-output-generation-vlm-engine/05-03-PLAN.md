<objective>
Integrate the two-pass VLM architecture into vlm_analyzer.py, replacing the hardcoded prompt with graph-backed prompt generation.

Purpose: This is the culmination of Phase 5. The VLMForehandAnalyzer.analyze_swing method currently uses a single hardcoded _FTT_SYSTEM_PROMPT for every analysis. This plan upgrades it to a two-pass flow: Pass 1 identifies symptom categories via a compact checklist, Pass 2 injects only the relevant diagnostic chains and subgraph context. This improves diagnostic accuracy while staying within token budgets. A single-pass fallback using the current prompt preserves backward compatibility when the knowledge graph is not available.

Output: Modified `evaluation/vlm_analyzer.py` with two-pass support, integration tests.
</objective>
