    1. `enrich_with_muscles(kg: KnowledgeGraph, muscle_map_path: Path)`:
       - Load _concept_muscle_map.json

    2. `annotate_vlm_features(kg: KnowledgeGraph)`:
       For each node by category:
       - **Symptoms** (category=="symptom"): The symptom description IS a VLM feature. Set vlm_features = [description]. These are things like "elbow flies out", "racket face open at contact".
       - **Techniques** (category=="technique"): Map to observable features based on keywords:
         * "rotation" -> ["visible trunk/hip rotation angle"]
         * "wrist" -> ["wrist angle at contact", "wrist lag visibility"]
         * "unit_turn" -> ["shoulder turn relative to hips", "racket position behind body"]
         * "contact_point" -> ["ball-racket contact position relative to body"]
         * "follow_through" -> ["racket path after contact", "finishing position"]
         * Use concept description to extract 1-3 VLM-observable features
       - **Biomechanics** (category=="biomechanics"): Map to joint angles and segment positions
       - **Drills** (category=="drill"): No VLM features needed (drills are instructions, not visible)

    3. `add_visible_as_edges(kg: KnowledgeGraph)`:
       For technique concepts that have VLM features, create "visible_as" edges connecting the technique to related symptom nodes. Look for symptom nodes whose description overlaps with the technique's VLM features (use simple keyword matching).

    4. `build_why_explanation(kg: KnowledgeGraph, concept_id: str) -> dict | None`:

       - concept: name, description

       - physics: from node's description + causal chain context
       - visible: from node's vlm_features or connected symptom nodes via "visible_as"
       Return as dict: {concept, muscles: [{name, function, role}], physics: str, visible_symptoms: [str]}
